# Part 1 : Linux File System Hierarchy
```
/ (root)
```
### What it contains:
  - The top-level directory of the Linux filesystem; all other directories start from here.
  - ls -l / (examples):
    - bin/
    - etc/
#### I would use this when , I want to understand the overall filesystem structure or navigate to any core directory.

```
/home
```
### What it contains:
- Home directories for normal users, where personal files and configs are stored.
- ls -l (examples):
    - /ubuntu
    - /devops
### I would use this when, I’m working with user files, scripts, or checking user-specific data.

```
/root
```
### What it contains:
  - The home directory of the root (superuser) account.
  - ls -l /root (examples):
      - .bashrc
      - scripts/
  ### I would use this when , I’m logged in as root and managing system-level scripts or configurations.

```
/etc
```
### What it contains:
  - System-wide configuration files for services and applications.
  - ls -l /etc (examples):
    - ssh/
    - passwd
 ### I would use this when , I need to configure services like SSH, NGINX, or system settings.

 ```
/var/log
```
### What it contains:
  - Log files generated by the system and running services.
  - ls -l /var/log (examples):
      - syslog
      - auth.log
### I would use this when , I’m troubleshooting errors, failures, or monitoring system behavior.
